# Паттерн Prototype

**Prototype** — это порождающий паттерн проектирования, который позволяет копировать существующие объекты, не вдаваясь в детали их реализации. Он создаёт новые объекты путём клонирования готового образца (прототипа), а не через вызов конструктора.

#### **Основная концепция**

Суть паттерна заключается в том, что:
- Создание нового объекта происходит через копирование существующего
- Код не зависит от конкретных классов копируемых объектов
- Объект сам отвечает за своё клонирование

#### **Преимущества использования**

1. **Экономия ресурсов**
   - Избегает дорогостоящих операций инициализации (запросы к БД, чтение файлов, сетевые вызовы)
   - Копирование часто быстрее создания с нуля

2. **Сокрытие сложности**
   - Не нужно знать, как устроен объект внутри
   - Не нужно передавать множество параметров в конструктор

3. **Динамическое добавление/удаление объектов**
   - Можно создавать копии объектов, чьи классы неизвестны на этапе компиляции
   - Удобно для работы с конфигурациями и шаблонами

4. **Упрощение создания "похожих" объектов**
   - Создал прототип → изменил пару полей → готов новый объект

#### **Ключевые компоненты**

1. **Prototype (Прототип)**
   - Интерфейс или абстрактный класс с методом `clone()`
   - Объявляет операцию клонирования

2. **ConcretePrototype (Конкретный прототип)**
   - Реализует метод клонирования
   - Создаёт копию самого себя

3. **Client (Клиент)**
   - Работает с прототипом через метод `clone()`
   - Не знает конкретный класс оригинала

#### **Лучшие практики**

1. **Различай поверхностное и глубокое копирование**
   - **Поверхностное** — копирует только ссылки на вложенные объекты
   - **Глубокое** — рекурсивно копирует все вложенные объекты
   - Выбирай в зависимости от ситуации

2. **Используй `copy` и `deepcopy` в Python**
   - Модуль `copy` уже предоставляет готовые механизмы
   - Не нужно изобретать велосипед

3. **Храни реестр прототипов (Prototype Registry)**
   - Для частого использования одних и тех же прототипов
   - Доступ по ключу: `registry.get("default_user").clone()`

4. **Учитывай циклические ссылки**
   - При глубоком копировании объектов с циклическими ссылками нужно быть осторожным
   - `copy.deepcopy()` обрабатывает такие случаи

#### **Пример реализации**

```python
import copy
from abc import ABC, abstractmethod

# Абстрактный прототип
class Prototype(ABC):
    @abstractmethod
    def clone(self):
        pass

# Конкретный прототип
class Address:
    def __init__(self, city: str, street: str):
        self.city = city
        self.street = street

    def __str__(self):
        return f"{self.city}, {self.street}"

class User(Prototype):
    def __init__(self, name: str, age: int, address: Address):
        self.name = name
        self.age = age
        self.address = address

    # Поверхностное копирование
    def clone(self):
        return copy.copy(self)

    # Глубокое копирование
    def deep_clone(self):
        return copy.deepcopy(self)

    def __str__(self):
        return f"User(name={self.name}, age={self.age}, address={self.address})"

# Использование
address = Address("Москва", "Тверская")
original = User("Анна", 25, address)

# Поверхностная копия
shallow_copy = original.clone()
shallow_copy.address.city = "СПб"  # изменится и у оригинала!

# Глубокая копия
deep_copy = original.deep_clone()
deep_copy.address.city = "Казань"  # оригинал не изменится

print(f"Оригинал: {original}")        # Address: СПб (изменился!)
print(f"Поверхностная: {shallow_copy}") # Address: СПб
print(f"Глубокая: {deep_copy}")         # Address: Казань
```

#### **Когда использовать**

- Создание объекта требует больших затрат ресурсов (БД, сеть, файлы)
- Нужно создать множество объектов, отличающихся незначительно
- Конструктор слишком сложный и требует много параметров
- Классы объектов неизвестны на этапе компиляции
- Нужно избежать наследования для создания "почти таких же" объектов

#### **Альтернативы и дополнения**

1. **Простой конструктор** — когда создание объекта дешёвое
2. **Factory Method** — когда нужно создавать объекты с нуля, а не копировать
3. **Реестр прототипов** — удобное хранилище для частого доступа к прототипам

## Пример использования Prototype для игровых персонажей

В играх часто создаётся множество однотипных врагов. Вместо того чтобы каждый раз задавать все характеристики с нуля, можно создать прототип и клонировать его.

```python
import copy
from typing import List

class Weapon:
    def __init__(self, name: str, damage: int):
        self.name = name
        self.damage = damage

class Enemy:
    def __init__(self, name: str, health: int, weapons: List[Weapon]):
        self.name = name
        self.health = health
        self.weapons = weapons

    def clone(self):
        """Поверхностное копирование (оружие будет общим)"""
        return copy.copy(self)

    def deep_clone(self):
        """Глубокое копирование (оружие будет своим у каждой копии)"""
        return copy.deepcopy(self)

    def __str__(self):
        weapons_names = [w.name for w in self.weapons]
        return f"{self.name} (HP: {self.health}, Weapons: {weapons_names})"

# Создаём прототипы
sword = Weapon("Меч", 15)
axe = Weapon("Топор", 20)

goblin_prototype = Enemy("Гоблин", 50, [sword])
ogre_prototype = Enemy("Огр", 150, [axe])

# Создаём армию через клонирование
army = []
for i in range(5):
    goblin = goblin_prototype.clone()
    goblin.name = f"Гоблин-{i+1}"
    army.append(goblin)

for i in range(2):
    ogre = ogre_prototype.clone()
    ogre.name = f"Огр-{i+1}"
    army.append(ogre)

# Изменяем здоровье одного из гоблинов — другие не пострадают
army[0].health = 30

for enemy in army:
    print(enemy)
```

### Как это работает

1. **Создаётся прототип** — базовый объект с типовыми характеристиками
2. **Клонирование** — метод `clone()` создаёт копию прототипа
3. **Настройка** — каждая копия может быть изменена индивидуально
4. **Экономия** — не нужно заново инициализировать сложные объекты

### Преимущества этого подхода

1. **Быстрое создание** — клонирование часто быстрее вызова конструктора
2. **Гибкость** — можно динамически добавлять новые типы врагов во время игры
3. **Простота** — не нужно хранить фабрики для каждого типа врага
4. **Разделение ответственности** — объект сам знает, как себя копировать

### Когда использовать этот подход

- **В играх** — создание врагов, пуль, частиц
- **В редакторах** — копирование элементов интерфейса или фигур
- **При работе с конфигурациями** — создание похожих настроек
- **В системах с кэшированием** — клонирование данных из кэша вместо повторных запросов
- **При создании сложных отчётов** — взять шаблон и подставить новые данные
